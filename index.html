WELCOME

Welcome to The Iron Yard UI Design course! I'm so excited to get to know you, to challenge you, and to give you the tools you need to be a successful designer. Although there are still a couple weeks until class starts, there is a lot to be done!

WHY PREWORK?

UI Design has a lot of different elements to consider. One point I’d like to make clear: UI Design is not just about making things look good, but rather its about juggling a bunch of elements such as: content, imagery, illustration, icon systems, typography, color theory and concepts to create a fully functioning and visually appealing design system. One of the biggest challenges will juggling all of those elements into a cohesive system and involves researching, experimenting, designing, coding, and testing. Learning these skills will take time and it won’t always be easy, and this pre-work will help introduce you to some of the ideas and skills that will form the foundation of your in-person learning experience.

This work should be completed before the first day of class. I will assume that you have completed this work before class. You don't need to memorize the information, but have a solid understanding of some of the areas that we will cover in more depth during the course.

OBJECTIVES

  • Be able to communicate technical accomplishments and struggles in designing, coding, and writing.
  • Be able to learn in contexts that aren't necessarily suited to your primary learning style.
  • Adapt to the workflows used by professional Web Designers (we've cover a few common ones in class).
  • Commit yourself fulling to learning as much as in can and completing all assignments to the best of your ability.
  • Describe the importance of mindset on learning and be able to analyze one's own type of mindset.

HTML AND CSS BASICS

HTML and CSS are the foundation of all the content of webpages, web sites, or web apps. However, we will only be spending about a week on HTML and CSS, so you should have a basic foundation before the class starts.

Complete the course: CodeAcademy – Web Track, which covers HTML and CSS. This should take you approximately 7-12 hours, depending on your prior experience.

MARKDOWN

Learn Markdown, a simple language that converts simple text to code through this tutorial.

TYPING

Believe it or not, a lot of time in the first weeks of coding and programming is spent fixing simple typing errors. It would not be ridiculous to suggest that many of you will spend 12-24 hours (or more) fixing typing errors during class.

If you don't know how to touch type (i.e. type without looking at the keyboard), try to spend some time each day working on touch typing (you can try KEYBR). Typing for programming is actually a good bit different than everyday typing, so getting some practice in could potentially save you a ton of time the first few weeks of class.
